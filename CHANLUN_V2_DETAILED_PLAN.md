## 项目概述

### 目标

开发一套**领先的趋势捕捉与预警系统**，实现：
- ✅ 抓住趋势起势 (提前 3-5 天入场)
- ✅ 看懂趋势衰减 (提前 3-5 天预警)
- ✅ 提前离场 (保住 80-90% 利润)

### 核心理念

```
传统缠论：背驰→确认→入场 (滞后，错过 10-15%)
改进系统：起势→持有→衰减→离场 (领先，抓住 70-80%)
```

### 预期效果

| 指标 | 当前 v5.3 | 目标 v2.0 | 提升 |
|------|---------|---------|------|
| **入场时机** | 背驰确认后 | 起势预警 | 提前 3-5 天 |
| **离场时机** | 反转确认后 | 衰减预警 | 提前 3-5 天 |
| **胜率** | 60-70% | 70-80% | +10-15% |
| **盈亏比** | 1.5-2.0 | 2.5-3.0 | +50% |
| **最大回撤** | 15-20% | 8-12% | -50% |
| **年化收益** | 20-30% | 35-50% | +75% |

## 开发原则

### 原则 1: 阶段验证

```
每个阶段完成后必须通过验证，才能进入下一阶段

开发 → 测试 → 验证 → 验收 → 下一阶段
        ↓
    不通过则返回修改
```

### 原则 2: 方案对比

```
关键技术决策前，必须提供至少 2 个方案

方案 A: 优缺点分析 + 实现成本 + 预期效果
方案 B: 优缺点分析 + 实现成本 + 预期效果
        ↓
    对比后选择最优方案
```

### 原则 3: 数据驱动

```
所有参数必须通过回测验证，不能凭经验设定

权重配置 → 回测优化 → 最优参数
阈值设定 → 回测优化 → 最优阈值
```

### 原则 4: 文档完整

```
每个阶段必须有完整文档

- 设计文档
- 测试报告
- 验收报告
- 用户手册
```

def detect_strength_decline(centers, segments):
    strengths = [calc_strength(seg) for seg in segments]
