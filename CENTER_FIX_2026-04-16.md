## 🐛 问题描述

### 用户发现的问题

> "买卖点的判断也是基于中枢的，目前你的判断逻辑是否包括中枢的确立判断？"

**问题核心**: 系统检测买卖点时，**没有使用中枢判断**，违反了缠论定义。

---

## 📐 缠论正统定义

### 第一类买卖点

```
中枢形成 → 离开中枢 → 背驰 → buy1/sell1

条件:
  1. 中枢已形成
  2. 价格离开中枢 (低于 ZD 或高于 ZG)
  3. MACD 背驰
```

### 第二类买卖点

```
buy1 → 反弹 → 回调 → buy2

条件:
  1. buy1 已确认
  2. 回调回到中枢内或中枢边界
  3. 不破 buy1 低点
```

### 第三类买卖点（新增）

```
中枢 → 突破 → 回测 → buy3

条件:
  1. 中枢已形成
  2. 价格突破中枢 (高于 ZG)
  3. 回测不破中枢上沿
```

## 📈 中枢数据结构

```python
Center:
  start_idx: int      # 中枢起始索引
  end_idx: int        # 中枢结束索引
  high (ZG): float    # 中枢上沿
  low (ZD): float     # 中枢下沿
  segments: List      # 组成中枢的线段
  
  方法:
    contains(price): 检查价格是否在中枢内
```
