### 问题描述

**现象**：同一标的同一级别同时出现买点和卖点警报

```
修复前 (14:00 扫描):
🟢 CNQ.TO 30m 第二类买点 @ $66.13
🔴 CNQ.TO 30m 第二类卖点 @ $66.13  ← 同一价格，矛盾！
```

### 根本原因

**违反缠论走势类型互斥原理**：
- 缺少趋势方向过滤
- 只检查分型距离，未检查当前笔方向
- 震荡市中价格同时接近前高和前低时误触发

### 修复方案

| 修复项 | 说明 | 理论依据 |
|--------|------|----------|
| **趋势方向过滤** | 买 2 只在上涨趋势，卖 2 只在下跌趋势 | 缠论走势类型互斥原理 |
| **信号互斥检查** | 同时出现时只保留置信度更高的 | 同一级别只能有一个主导趋势 |
| **阈值优化** | 30m: 2.5% → 1.5%，添加最小距离过滤 | 避免震荡市误触发 |

### 缠论理论应用

**核心原理**：走势类型互斥原理

> "任何级别的任何走势，只可能有三种类型：上涨、下跌、盘整。"
> "这三种类型是互斥的，同一级别同一时间只能有一种走势类型。"

**算法实现**：
```python
# 获取当前趋势方向
last_pen = pens[-1]
current_trend = 'up' if last_pen.is_up else 'down'

# 第二类买点：只在上涨趋势中
if current_trend == 'up':
    # 买点逻辑...

# 第二类卖点：只在下跌趋势中
if current_trend == 'down':
    # 卖点逻辑...
```
