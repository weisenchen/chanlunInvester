## 1. 编码前思考 (Think Before Coding)

### 目标

优化 Phase 1-4 参数，提升整体性能：
- Phase 1: 增加样本数量 (20→100)
- Phase 2: 产生回测信号 (0→50+)
- Phase 3: 提升利润保住率 (24%→80%)
- Phase 4: 提升综合准确率 (55.9%→80%)

### 核心问题

**Phase 1: 样本数量不足 (20 个)**
- 原因：只有 TSLA 产生信号
- 解决：增加高波动股票

**Phase 2: 回测无信号**
- 原因：中枢检测条件严格
- 解决：降低中枢检测门槛

**Phase 3: 利润保住率偏低 (24%)**
- 原因：退出策略未优化
- 解决：优化止盈止损逻辑

**Phase 4: 综合准确率偏低 (55.9%)**
- 原因：简单平均可能不是最优
- 解决：优化权重配置

### 技术方案对比

#### 方案 A: 增加股票池 (推荐)

```python
SYMBOL_LIST = [
    # 原有高波动股票
    'TSLA', 'NVDA', 'AMD', 'COIN', 'PLTR', 'SMR', 'IONQ', 'RKLB',
    # 新增高波动股票
    'MARA', 'RIOT', 'COIN', 'HOOD', 'SOFI', 'LCID', 'RIVN', 'NIO',
]
```

**优点**: 
- 快速增加样本
- 不改变核心逻辑
- 便于实施

**缺点**: 
- 需要更多数据获取
- 回测时间增加

---

#### 方案 B: 降低检测门槛

```python
# 中枢检测门槛从 3 降到 2
self.center_detector = CenterDetector(min_segments=2)
```

**优点**: 
- 产生更多信号
- 不增加股票

**缺点**: 
- 可能降低信号质量
- 需要重新验证

---

#### 方案选择

**决策**: 采用**方案 A + B 组合**

**理由**:
1. 增加股票池快速增加样本
2. 降低检测门槛产生更多信号
3. 两者结合效果最佳
