### 2026-04-16 17:30 EDT - Phase 1 信号检测方案选择

**决策**: 采用**方案 A (多因子加权)**

**背景**:
Phase 1 趋势起势检测需要选择信号检测方法

**备选方案**:

#### 方案 A: 多因子加权
```python
probability = (
    breakout * 0.25 +
    momentum * 0.20 +
    volume * 0.20 +
    resonance * 0.15 +
    ma * 0.10
)
```

**优点**:
- 简单易懂
- 权重可优化
- 易于解释

**缺点**:
- 线性组合，可能忽略非线性关系
- 权重需要大量回测

**实现成本**: 低 (已完成)  
**预期胜率**: 65-70%

---

#### 方案 B: 机器学习分类器
```python
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier()
model.fit(X_train, y_train)
probability = model.predict_proba(X_test)
```

**优点**:
- 可捕捉非线性关系
- 自动学习特征重要性

**缺点**:
- 需要大量标注数据
- 黑盒模型，难以解释
- 过拟合风险

**实现成本**: 高 (2 周)  
**预期胜率**: 70-75%

---

**决策理由**:
1. 当前数据量不足，机器学习效果不确定
2. 多因子加权已足够，胜率目标 65% 可达到
3. 模型可解释性强，便于调优
4. 开发成本低，可快速验证

**备注**: Phase 6 回测框架完成后，可重新评估方案 B

**决策者**: ChanLun AI Agent  
**状态**: ✅ 已实施

### 2026-04-16 17:15 EDT - 分支管理策略

**决策**: 采用**双分支策略**

**背景**:
需要备份 v5.3 完整版，同时开发 v2.0 改进版

**备选方案**:

#### 方案 A: 单分支开发
- 直接在 main 分支开发
- 风险：可能破坏现有功能

#### 方案 B: 双分支策略
- `chanlun-origin`: v5.3 完整版备份

- 版本隔离，降低风险

- 合并可能复杂

**决策理由**:
1. v5.3 是稳定版本，需要保留
2. v2.0 是重大改进，风险较高
3. 分支隔离便于测试和回滚

### 2026-04-16 17:00 EDT - 阶段验收原则

**决策**: 采用**严格阶段验收**原则

**背景**:
需要确定开发流程管理方式

**原则**:
1. 每个阶段必须通过验证才能进入下一阶段
2. 不通过则返回修改
3. 验收标准量化

**验收流程**:
```
开发完成 → 单元测试 → 集成测试 → 回测验证 → 代码审查 → 文档审查 → 验收通过
                                      ↓
                                  不通过则返回修改
```

**决策理由**:
1. 避免问题累积到后期
2. 确保每一步都扎实
3. 便于问题定位和修复

**决策者**: ChanLun AI Agent  
**状态**: ✅ 已实施

### 2026-04-16 16:30 EDT - 中枢判断逻辑

**决策**: **买卖点必须结合中枢判断**

**背景**:
用户指出"买卖点的判断也是基于中枢的"，当前系统没有使用中枢判断

**变更内容**:
- buy1: 背驰 + 离开中枢
- buy2: buy1 确认 + 回测中枢
- sell1: 背驰 + 离开中枢
- sell2: sell1 确认 + 回测中枢
- buy3: 突破中枢 + 回测确认
- sell3: 跌破中枢 + 回测确认

**决策理由**:
1. 符合缠论正统定义
2. 提高信号准确性
3. 过滤假信号

**决策者**: ChanLun AI Agent  
**状态**: ✅ 已实施 (v5.3)

### 2026-04-16 16:00 EDT - buy1 检查逻辑

**决策**: **第二类买卖点必须检查第一类买卖点已确认**

**背景**:
用户指出"第二买卖点是建立在第一买卖点已发生的条件上的"

**变更内容**:

- sell2 检测前检查 sell1 是否已确认

3. 提高信号质量

**效果**:

- 修复后：1 只共振信号 (100% 真信号)
