## 1. 编码前思考 (Think Before Coding)

### 目标

开发趋势反转预警模块，实现：
- 多信号共振检测
- 提前 3-5 天预警反转
- 识别率≥75%

### 核心概念

**趋势反转**: 上涨→下跌 或 下跌→上涨的转折点

**检测维度**:
1. **多级别背驰共振** (35%) - 日线 +30m+5m 同时背驰
2. **第三类买卖点失败** (20%) - buy3/sell3 后迅速反转
3. **中枢升级完成** (15%) - 小中枢升级为大中枢
4. **先行指标背离** (15%) - MACD/成交量先行背离
5. **市场情绪反转** (15%) - 新闻/资金流反转

### 技术方案对比

#### 方案 A: 多信号共振 (推荐)

```python
def warn_reversal(signals):
    # 多个信号同时触发
    signal_count = sum(1 for s in signals if s.triggered)
    
    if signal_count >= 3:  # 至少 3 个信号触发
        return True  # 反转预警
```

**优点**: 
- 容错性强
- 减少误报
- 符合缠论"多重确认"理念

**缺点**: 
- 需要多个信号同时触发
- 可能错过早期信号

---

#### 方案 B: 加权评分

```python
def warn_reversal(signals):
    # 加权评分
    score = sum(s.weight * s.strength for s in signals)
    
    if score >= 0.7:  # 综合评分≥70%
        return True  # 反转预警
```

**优点**: 
- 考虑信号强度差异
- 更精细的评估

**缺点**: 
- 权重设定复杂
- 需要回测优化

---

#### 方案选择

**决策**: 采用**方案 B (加权评分)** 为主，方案 A 为辅

**理由**:
1. 不同信号重要性不同 (多级别背驰 > 单一信号)
2. 加权评分更精细
3. 与 Phase 1/2 保持一致
4. 方案 A 作为过滤条件 (至少 2 个信号)
