## 1. 编码前思考 (Think Before Coding)

### 目标

开发趋势衰减监测模块，实现：
- 实时监测趋势衰减
- 提前 3-5 天预警
- 准确率≥80%

### 核心概念

**趋势衰减**: 上涨/下跌趋势的动能逐渐减弱

**检测维度**:
1. **力度递减** (30%) - 离开中枢的线段力度减弱
2. **中枢扩大** (20%) - 中枢区间变大，震荡加剧
3. **时间延长** (20%) - 中枢形成时间变长
4. **量价背离** (15%) - 价格上涨但成交量萎缩
5. **多级别背离** (15%) - 大级别涨小级别跌

### 技术方案对比

#### 方案 A: 阈值判断 (推荐)

```python
def detect_strength_decline(centers, segments):
    strengths = [calc_strength(seg) for seg in segments]
    decline_rate = (strengths[-1] - strengths[-2]) / strengths[-2]
    
    if decline_rate < -0.3:  # 力度下降 30%
        return True  # 衰减预警
```

**优点**: 
- 简单直接
- 易于调优
- 符合缠论定义

**缺点**: 
- 阈值需要回测确定

---

#### 方案 B: 趋势线突破

```python
def detect_trendline_break(prices):
    trendline = fit_trendline(prices)
    if prices[-1] < trendline[-1]:
        return True  # 衰减预警
```

**优点**: 直观
**缺点**: 趋势线拟合主观性强

---

**决策**: 采用**方案 A (阈值判断)** 为主，方案 B 为辅

**理由**:
1. 力度递减是缠论核心概念
2. 阈值可通过回测优化
3. 趋势线突破可作为辅助信号
