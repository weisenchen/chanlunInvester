## 📚 planning-with-files 技能概述

### 核心理念

> **Work like Manus** — the AI agent company Meta acquired for **$2 billion**.

使用**持久化 markdown 文件**进行规划、进度跟踪和知识存储。

### 核心价值

1. **持久化规划**: 使用 markdown 文件记录计划，不依赖会话上下文
2. **进度跟踪**: 实时更新进度，保持开发连续性
3. **知识积累**: 记录决策、经验、教训，形成知识库
4. **会话恢复**: 即使会话中断，也能从规划文件恢复

## 📊 工作流对比

### 传统方式

```
用户请求 → 开发 → 完成
          ↓
      无记录，下次重新开始
```

**问题**:
- 会话中断后丢失上下文
- 无法跟踪长期进度
- 决策无记录，重复犯错

---

### planning-with-files 方式

```
用户请求 → 创建规划文件 → 开发 → 更新进度 → 完成
              ↓              ↓         ↓
          目标明确       进度可见    决策记录
```

**优势**:
- ✅ 会话恢复：从规划文件恢复上下文
- ✅ 进度可见：随时查看完成度
- ✅ 决策记录：避免重复犯错
- ✅ 知识积累：形成可复用的知识库

## 🎓 经验总结

### 1. 规划文件价值

```
无规划: 开发→忘记→重新开发 (循环)
有规划: 开发→记录→继续开发 (连续)
```

**收获**:
- 会话中断后能快速恢复
- 进度清晰可见
- 决策有记录可查

---

### 2. 待办事项分级

```
🔴 紧急 (今日完成)
🟡 重要 (本周完成)
🟢 后续 (未来阶段)
```

**收获**:
- 优先级清晰
- 避免遗漏重要事项
- 便于时间管理

---

### 3. 决策日志价值

```
决策前：方案 A vs 方案 B 对比
决策后：记录理由和预期效果
复盘时：验证决策是否正确
```

**收获**:
- 避免重复讨论同一问题
- 决策理由可追溯
- 便于后续优化
