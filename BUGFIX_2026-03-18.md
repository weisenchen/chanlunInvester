# 🐛 Bug 修复报告 - 价格显示错误

**修复日期**: 2026-03-18 11:00 EDT  
**问题发现**: 用户报告 XEG.TO 价格显示为 $5.75 而非 $25.75

---

## 🔍 问题分析

### 症状
```
❌ 错误显示:
💰 价格：$5.75

✅ 实际价格:
💰 价格：$25.75
```

### 根本原因

**Shell 变量扩展问题** - 在 shell 命令中，`$` 符号后跟数字会被 shell 尝试解释为变量：

```bash
# 问题代码
cmd = f'openclaw message send --target "telegram:{TELEGRAM_CHAT_ID}" -m "{safe_message}"'

# 当消息包含 "$25.75" 时，shell 会尝试将 $25 解释为变量
# 导致 "$25.75" 变成 ".75" 或 "5.75"
```

---

## ✅ 修复方案

### 方案 1: 使用 `USD` 前缀代替 `$` 符号 ✅ (已采用)

```python
# 修复前
price_str = f"${signal['price']:.2f}"  # $25.75

# 修复后
price_str = f"USD {signal['price']:.2f}"  # USD 25.75
```

### 方案 2: 改进 shell 转义

```python
# 使用 shlex 模块正确转义
import shlex
safe_message = message.replace("'", "'\"'\"'")
cmd = f"openclaw message send --target 'telegram:{TELEGRAM_CHAT_ID}' -m '{safe_message}'"
```

## 🛡️ 预防措施

### 未来开发注意事项

1. **避免在 shell 命令中使用 `$` 符号**
   - 使用 `USD` 代替 `$`
   - 或使用其他货币符号如 `C$` (加元)

2. **Shell 命令参数处理最佳实践**
   ```python
   # ❌ 不安全
   cmd = f'echo "$price"'
   
   # ✅ 安全 - 使用单引号
   cmd = f"echo '{price}'"
   
   # ✅ 更安全 - 使用 subprocess 列表
   subprocess.run(['echo', price])
   ```

3. **测试时检查特殊字符**
   - `$` (变量)
   - `` ` `` (命令替换)
   - `\` (转义)
   - `"` 和 `'` (引号)
